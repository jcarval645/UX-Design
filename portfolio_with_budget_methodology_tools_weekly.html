Project Title & Description

Project Title: Personal Development Portfolio: UX/UI Design

Design a prototype to identify a flaw within the university’s mobile app, GETmobile. The final version will be a professional portfolio showcasing the process and an interactive Figma prototype.

Project Goal

To develop professional UX/UI design competency by executing a complete design cycle—conducting user research, creating prototypes in Figma, and performing usability testing—to identify and solve a usability problem in the GETmobile app. This work will be documented in a professional portfolio to demonstrate capabilities to future employers.

Vision, Mission, & Strategy Alignment

Personal Mission Connection: This project reflects my mission to “dedicate myself to gaining technical and analytical skills.” It will allow me to gain technical skills like Figma and prototyping, and analytical skills such as user research, to build my future by developing skills that prove my capability.

Vision Alignment: Creating a professional portfolio gives me an idea of what it is like to work in the tech industry to solve real-life problems. It also allows me to make myself a stronger job applicant.

Strategic Goal Support: This project helps me to achieve one of my strategic goals: Improve technical skills. This project is basically a hands-on tutorial. I will watch videos and actively use Figma to build a prototype.

Business Case & Justification

Professional Value:

  • Skills Gained: I will graduate with new skills in user research, prototyping, and usability testing, complementing my CIS degree.
  • Improving My Portfolio: This portfolio shows employers that I can apply my knowledge to real-life problems and deliver an outcome.

Personal Value:

  • Clarity: It provides an example of exploring a potential career path, reducing the overwhelming options in the tech field.
  • Learning Outcome:
    • Technical skills: Figma proficiency, prototyping, and portfolio development.
    • Industry Knowledge: Gaining an understanding of user-centered design processes and how they incorporate with technology.
High-Level Scope

In Scope for Semester-Long Project

  • Write a short report regarding the survey I put out on the app’s most significant problems.
  • Pull out one problem that classmates often mention in the survey.
  • Create a simple plan of multiple solutions.
  • Create a prototype on Figma that will fix the issue.
  • Collect feedback from testing the prototype with a few classmates, and write a report on that.
  • Create an online portfolio page displaying the project.

Roadmap Items - Up to 6 Months After Semester Ends

  • Expanding the prototype to include other things besides fixing the issue that I fixed.
  • Creating a second round of user testing.
  • Creating a style guide to ensure consistency for all screens.
  • Incorporating code such as HTML.
  • Use portfolio in internship/job applications

Out of Scope

  • Building up a real app that we can download.
  • Releasing the app on the Apple App Store or the Google Play Store.
  • Fixing the entire GETmobile app.
  • Collecting data from hundreds of people.

Deliverables and Work Breakdown Structure

Definition of Done

Project Brief & Problem Statement
  • Survey receives 10+ responses from GETmobile users
  • Top 3 usability issues identified with data
  • One primary problem selected with clear justification
  • Problem statement is specific and user-focused
  • One-page brief completed in Google Docs with data visualization
  • Document exported and saved as PDF
Current User Map
  • All users' steps are documented from app
  • Capture screenshots of each step
  • Take notes on problem areas
  • Highlight issues with color
  • Create a map on FigJam
  • Map exported as PDF
Prototype
  • Wireframe completed for all screens
  • High-fidelity mockup created with visual style guide
  • Create an interactive prototype
  • Address the issue from Deliverable 1 with a solution
  • Ensure all UI components are consistent
  • The prototype is ready for testing
Test Report
  • 3 usability test sessions with classmates
  • Test 3 test scripts and document
  • Take notes based on observations and issues
  • Summarize my findings, recommendations, etc
  • Write a one-page summary report
  • Test report exported as PDF
Portfolio
  • All 4 deliverables included as a case study
  • Figma prototype linked with clear instructions
  • Visualizations are high quality and formatted
  • Page is designed well and responsive to other devices
  • Portfolio URL is public

Project Methodology Explanation

The diagram shows the four phases: Discover, design and prototype, test, and lastly deliver. It is providing a clear structure for my timeline. The diagram contains two iterative cycles.

The first cycle covers:

  • Creating a wireframe
  • Get feedback from classmates
  • Go over it until I am satisfied with the design

The second cycle covers:

  • Test high-fidelity prototype with users
  • Analyze data
  • Make revisions based on analyzation

Each phase ends with a deliverable as outlined on my Work Breakdown Structure sheet. The design visually highlights how feedback is key to the project, leading to revisions and improvements, ensuring the final portfolio is perfect.

Project Tool Selection Explanation

After evaluating three project tool platforms, I decided to proceed with Smartsheet. Even if my methodology is hybrid iterative rather than predictive, Smartsheet is still the best fit for this project. An advantage of selecting this platform is the ability to upload my Excel WBS sheet directly, removing the process of reentering all of my tasks, which can take up a lot of my time. Smartsheet is also perfect for me personally, since I am not an organized person, and a checkbox will not do the job for me. It allows me to view the entire project timeline.
